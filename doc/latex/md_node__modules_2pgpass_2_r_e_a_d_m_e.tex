\hypertarget{md_node__modules_2pgpass_2_r_e_a_d_m_e_autotoc_md1554}{}\doxysection{\texorpdfstring{Description}{Description}}\label{md_node__modules_2pgpass_2_r_e_a_d_m_e_autotoc_md1554}
This module tries to read the {\ttfamily \texorpdfstring{$\sim$}{\string~}/.pgpass} file (or the equivalent for windows systems). If the environment variable {\ttfamily PGPASSFILE} is set, this file is used instead. If everything goes right, the password from said file is passed to the callback; if the password cannot be read {\ttfamily undefined} is passed to the callback.

Cases where {\ttfamily undefined} is returned\+:


\begin{DoxyItemize}
\item the environment variable {\ttfamily PGPASSWORD} is set
\item the file cannot be read (wrong permissions, no such file, ...)
\item for non windows systems\+: the file is write-\//readable by the group or by other users
\item there is no matching line for the given connection info
\end{DoxyItemize}

There should be no need to use this module directly; it is already included in {\ttfamily node-\/postgres}.\hypertarget{md_node__modules_2pgpass_2_r_e_a_d_m_e_autotoc_md1555}{}\doxysection{\texorpdfstring{Configuration}{Configuration}}\label{md_node__modules_2pgpass_2_r_e_a_d_m_e_autotoc_md1555}
The module reads the environment variable {\ttfamily PGPASS\+\_\+\+NO\+\_\+\+DEESCAPE} to decide if the the read tokens from the password file should be de-\/escaped or not. Default is to do de-\/escaping. For further information on this see \href{https://github.com/postgres/postgres/commit/8d15e3ec4fcb735875a8a70a09ec0c62153c3329}{\texttt{ this commit}}.\hypertarget{md_node__modules_2pgpass_2_r_e_a_d_m_e_autotoc_md1556}{}\doxysection{\texorpdfstring{Tests}{Tests}}\label{md_node__modules_2pgpass_2_r_e_a_d_m_e_autotoc_md1556}
There are tests in {\ttfamily ./test/}; including linting and coverage testing. Running {\ttfamily npm test} runs\+:


\begin{DoxyItemize}
\item {\ttfamily jshint}
\item {\ttfamily mocha} tests
\item {\ttfamily jscoverage} and {\ttfamily mocha -\/R html-\/cov}
\end{DoxyItemize}
